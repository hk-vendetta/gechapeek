# RefactorGuard Screenshot Plan

Goal: communicate value in under 10 seconds per image.

Recommended demo project:
- Unity Open Project 1: Chop Chop
- Source: https://github.com/UnityTechnologies/open-project-1

Pre-capture setup:
- Use this public sample project only (no private code or assets)
- Import RefactorGuard scripts into Assets/Editor/RefactorGuard
- Open the project once and let all imports finish
- Run one full Scan Project in RefactorGuard before taking screenshots
- Set Game and Scene tabs aside to maximize Inspector and Project visibility

## Shot 1 - Hero value
Title text: Stop breaking your project during refactors

Frame:
- RefactorGuard window open
- Single asset selected
- Caution banner visible
- Incoming list showing multiple assets

Purpose:
- Communicate core problem and immediate payoff

## Shot 2 - Single asset report

Purpose:

## Shot 6 - Export and team workflow
Title text: Export JSON and CSV for PRs and release review

Frame:
- Export buttons visible

Purpose:
- Show team-ready output, not just in-window analysis

## Visual guidance
- Use a medium-size sample project with enough dependencies to show realistic counts
- Prefer readable path names over lorem-style assets
- Keep Unity theme consistent across all shots
- Crop to avoid dead space; emphasize actionable UI zones
- Avoid red-only visuals in every shot; alternate with high, medium, and safe states
